# Add .conclaude.yml to Default Uneditable Files

## Why

Users should be protected from accidentally allowing Claude Code to modify conclaude's own configuration file. Since conclaude acts as a guardrail, allowing AI to freely modify its configuration creates a security risk where the AI could disable protections or alter validation rules. Adding `.conclaude.yml` to the default uneditable files list ensures consistent protection out-of-the-box.

## What Changes

- Add `.conclaude.yml` (and `.conclaude.yaml`) to the `rules.uneditableFiles` array in the default configuration
- Update comments in default configuration to document this protection
- Update schema documentation to reflect this security best practice
- No changes to the configuration schema itself (uses existing `uneditableFiles` array)

## Impact

- **Affected specs**: New capability - `configuration-defaults`
- **Affected code**:
  - `src/default-config.yaml` - Add `.conclaude.yml` and `.conclaude.yaml` to the default `rules.uneditableFiles` array
  - Documentation comments in default config file
- **Breaking changes**: None - this only affects the default template generated by `conclaude init`, not existing configurations
- **User experience**: Better security posture by default, users can still override if needed
